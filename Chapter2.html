Which of the following names a technique in which DNS requests for bad domains are rerouted to a custom IP of choice or 0.0.0.0?

correct answer: DNS sinkhole.

  • DNS tunneling
  • DNS whitelisting
  • DNS zone transfer
  • DNS sinkhole
Explanation

DNS sinkhole is the process by which requests to bad domains are sent to 0.0.0.0 or an IP of one’s choice. While a DNS sinkhole blocks access to bad domains, it is equally important that you are notified a machine is attempting to access a known bad domain. Using DNS query logging and looking for the IP address being used to sinkhole domains gives you exactly that.

Question 13 of 20

Which of the following statements regarding traditional service log collection is true?

correct answer: It is easy to comprehend and simple to set up; however, there are lots of systems to manage..

  • It requires network visibility and log formats are consistent.
  • Multiple application logs are created from a single application.
  • As a benefit, it covers all unknown and un-configured systems.
  • It is easy to comprehend and simple to set up; however, there are lots of systems to manage.
Explanation

Traditional log collection using an agent is easy to comprehend and deploy. It does not require any form of network mirroring to work. A drawback to traditional logging is that you have to deploy agents to many systems, making configuration management potentially difficult. Having a log agent does not mean that you can simply collect logs. Changes may be required to services (such as DNS) before the log agent is able to pick up the log. Because of this, there can be issues with collecting certain logs using traditional agent/agentless methods. Traditional logging does not require network visibility, but is blind to unknown or un-configured systems.

Question 15 of 20

What are the two main methods for collecting service logs?

correct answer: Traditional and network extraction.

  • Traditional and network extraction
  • Agent and agentless
  • Wiretap and port mirror
  • Traditional and modern
Explanation

The two main methods of collecting logs are traditional and network extraction. The traditional method for collecting service logs is to install agents on all servers or enable syslog when using an appliance. With network extraction, one uses a network monitoring system to look at packets and create logs as they are seen on the network. Thus, instead of multiple agents on various application servers, a single network monitoring host can often see and generate all logs from one spot.

Question 17 of 20

When reviewing WHOIS command output for a domain, which of the following fields can indicate that the domain in question is fake and has been cloned as part of a phishing campaign against a long established business?

correct answer: Creation date.

  • Multiple domain name servers
  • Creation date
  • Registrar WHOIS server
  • Expiration date
Explanation

Monitoring domain age is an interesting technique. Domains are registered with ICANN1; part of this registration record keeps track of when a domain was created. Querying WHOIS lookups can retrieve this information. For example, the Linux command "whois sec555.com | grep 'Creation Date' | grep Z | cut -d’:’ -f2 | cut -d' ' -f2" returns the creation date for sec555.com. This technique would not scale well if applying against every DNS query log. Instead, it works best when filtering out Alexa domains and running it only once against new domains, if possible.

Question 18 of 20

A SIEM administrator wants to generate a list of domain permutations that can be fed into their SIEM to enhance potential phishing attack detections. What tool could assist with this list creation?

correct answer: dnstwist.

  • DomainXpander
  • dnstwist
  • SIEM Admin Tool
  • phishList
Explanation

dnstwist can generate domain permutations, thus enhancing potential attack detection. The list of permutations can be fed to a SIEM as a lookup table. Phishing attacks can combine multiple techniques, such as: homograph attacks, character replacement attacks, and character transposing—where a phishing domain may have more than two characters changed. As an example, a homograph attack can use valid Unicode character sets (such as Cyrillic, Greek, or Armenian) to completely change every character in a domain, while the domain appears the same under a microscope. Another approach to detecting this is using a domain permutation engine to identify all possible and valid permutations of a domain.

When looking for malicious activity, why is there value in hunting through DNS NXDOMAIN response activity?

correct answer: Identifies domain generation algorithms.

  • Identifies domain generation algorithms
  • Identifies applications with DNS leaks
  • Identifies end users using unapproved cloud storage services
  • Identifies clients acting as C2
Explanation

Hunting through NXDOMAIN responses can highlight domain generation algorithms (DGAs) that were not registered. NXDOMAIN (non-existent domain) responses occur in normal environments, but their frequency is low. There are situations where NXDOMAIN responses may occur (such as when a typo is made or when Chrome is looking for DNS hijacking), but they can also signal that hosts are conducting DNS queries related to DGAs that were not registered.
