Question 3 of 20

The security engineering team at a large enterprise proposes deployment and support of a packet capture solution. Which of the following architectures would support the ability to conduct quick searches without severely impacting network bandwidth?

correct answer: A decentralized architecture with centralized querying. selected Responses

Explanation

A decentralized architecture reduces bandwidth, but centralized querying allows analysts to perform searches from a common console in their operations center.

The value of a good PCAP should never be underestimated; the level of detail provided is just too good to pass up. The problem is that packet captures are a recording of what actually happened. A 1 GB file sent across the network will be fully captured during a packet capture. Trying to shove packet captures across all locations to a centralized location simply will not work.

Fortunately, there are multiple decentralized solutions that capture packet data locally and send metadata to a central location. The metadata can then be used to search for and pull back selective packet captures for analysis. A few SIEM vendors offer this, but it usually comes at a premium cost.

Question 4 of 20

What type of enrichment can help identify the criticality of an alert based on the asset type?

correct answer: Tags. selected Responses

Explanation

Asset tags are not to be underestimated. While simple to apply, the context provided is vast. A DNS zone transfer alert is when an attacker tries to dump the contents of DNS for reconnaissance purposes. Assuming this kind of attack is performed against a workstation and tagging identifies the destination system as a workstation, an analyst can quickly mark the attack as attempted reconnaissance and move on. If the attack is against an actual DNS server and a tag identifies it as an internal DNS server, then the analyst knows to investigate. Again, though tags are simple in nature when applied correctly, they can have a dramatic effect on the analysis process.

correct answer: NIDS—out of band; NIPS—inline. selected Responses

Explanation

A network intrusion detection system (NIDS) is usually deployed out of band; traffic is sent to IDS through a Tap or SPAN port, which mirrors all the traffic passing through all ports. A network intrusion prevention system (NIPS) is deployed inline to stop the malicious traffic.

Explanation

Suricata supports an output format called EVE. This output option provides native JSON logging. Moreover, it provides many additional fields that can be enabled or disabled with ease. Each option within EVE is set to either yes or no to enable or disable. EVE logging will automatically extract information such as HTTP methods and hostnames, as long as the alert involves the corresponding application. EVE logging also allows granular configuration and multiple output files. For example, maybe you want to output to alert.json for Suricata alerts but also want to enable some of Suricata's extra capabilities that log all DNS requests to dns.json.

Question 17 of 20

Which open-source project can be used to write SIEM rules in a generic language to decouple rule logic from SIEM vendor and field names?

correct answer: Sigma. selected Responses

Explanation

Sigma is an open-source project that provides a generic language for writing SIEM rules. It allows an organization to write rules in Sigma format then convert them (with a tool called sigmac) into rules for a specific SIEM product. The key advantage here is that rules can become portable, thus more likely to be shared. To that effect, the Sigma project has a rules folder that contains many pre-written community rules that also map to MITRE techniques and categories.

Question 18 of 20

What possibility should analysts consider when submitting files or URLs to public malware sandboxes, such as VirusTotal?

correct answer: Attackers may be notified of the submission.. selected Responses

Explanation

Analysts should take precautions when submitting files or URLs to public malware sandboxes (like VirusTotal) because attackers may identify the submission. With the correct subscription, users can monitor the files, hashes, and URLs submitted to public sandboxes. This allows them to identify who submitted the request and take appropriate steps to change their methods or tactics. Analysts should make every attempt to analyze files offline or obfuscate their searches.

A new security analyst joins the security team. When teaching the analyst the difference between events and alerts, how should an incident be defined?

correct answer: A confirmed unwanted or unauthorized event. selected Responses

Explanation

Before diving into alerts and alert systems, it is important to share a common definition of an alert. If certain events are monitored because they may be unwanted or unauthorized, then those events are considered alerts, as they are meant to be investigated. Incidents, on the other hand, are confirmed unwanted or unauthorized events—basically, something bad has happened and it is time to do something about it.

Question 20 of 20

An analyst is reviewing IDS alerts in the SIEM and notices that there are over 100,000 alerts in the past week for "RDP connection confirm" between internal hosts expected to generate a high volume of RDP connections. What is the most performance efficient method of tuning this IDS alert?

correct answer: Modify or disable the single IDS alert rule on the IDS.. selected Responses

Explanation

The rule should either be modified or disabled on the IDS. There is no point in sending data to the SIEM only to drop it when the NIDS can suppress the alert. The reason behind this is it will improve the performance of both the SIEM and any other security device involved. An exception should be made when filtering at the SIEM is significantly easier than dealing with the other device. Some systems do not have filtering capabilities or are difficult to manage. In this case, make a management decision on whether or not to deal with it at the SIEM level.
