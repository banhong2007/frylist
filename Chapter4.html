What feature of Nmap is used to identify device types, services, and operating systems on a network?

correct answer: Fingerprint database. selected Responses
Explanation

Nmap can be used as a basic port scanner that sends crafted network packets to see if a port is open on a device. In taking things a step further, it sends many crafted packets to identify running services, even the underlying operating system and device type of an IP address. When reviewing DHCP log files of devices connected to a network, what part of the MAC address helps identify unauthorized devices based on the manufacturer of the device?

correct answer: OUI. selected Responses
Explanation

Using the organizationally unique identifier (OUI), it is possible to distinguish between corporate assets and unauthorized unknown devices. The first six hexadecimal characters of the MAC address on a device make up the OUI, which specifies the vendor used to manufacture the device. For instance, if an organization only uses Dell systems and log files show a MAC address with the OUI of an Apple device, this signals an unauthorized device to be investigated.

Which of the following HTTPS network flows is suspicious and should be investigated further?

correct answer: Internal database server uploading 5GB to the Internet and downloading 15MB from the same host. selected Responses
Explanation

Data exfiltration often manifests as one large upload or multiple decent-sized uploads. One way to look for this is to use the previous outbound byte monitoring. Command and control (C2), on the other hand, constantly checks in or out with a system. This can be as fast as every second or spread out to once a day. The commands received by the victim represent a download. The output or data sent back to the C2 server would be upload. As a result, C2 tends to be more upload than download. Looking for high connection counts with abnormal upload percentages specific to HTTP and HTTPS connections can help identify data exfiltration.

Question 10 of 20

Which of the following can be used to extend Windows process logging with a high level of granularity?

correct answer: Sysmon. selected Responses
Explanation

Sysmon supports granular capabilities to include or exclude events and thus can be used for monitoring. If agentless log collection is in use, calls can still be made to retrieve Sysmon logs, as they are in a standard Windows event channel. Even with certain log agents, centrally maintaining and changing configurations can be difficult. With Sysmon, this is a simple GPO change. New versions of Sysmon use the registry and automatically load changes. When tuned properly, Sysmon has little to no impact on the end-user experience. The filtering capabilities are extremely granular. For example, you can collect only processes that run with an integrity level of medium or lower. This would only monitor processes that are likely executed by end users or that have not been optimized for protection.

Question 18 of 20

Network scanning is considered an effective way to discover systems present on a network. What is one major problem with the use of active scanning?

correct answer: Devices that ignore traffic coming from scanners will not be discovered.. selected Responses
Explanation

Devices that use technologies like host-based firewalls or host-based intrusion detection are regularly set to drop traffic from hosts scanning a network. As a result, active scanning is dependent on the devices to be discovered responding to the probe traffic. Best results for discovery of a network are obtained using a combination of passive and active scanning.

Of the most critical drawbacks to active discovery, it requires the end device to respond to the scanner. If a device is on and receives an ICMP echo request packet but does not respond with an ICMP echo reply, then it is likely to be skipped by a scanner.

A security analyst wants to leverage Zeek to extract the software components used on the corporate network. Which of the following log files provides this information?

correct answer: software.log. selected Responses
Explanation

Provided by Zeek, software.log attempts to record software components that can be used to identify the underlying system. One of the reasons this log is so helpful is that it does not repetitively log the same thing over and over. For example, if an iPad is discovered browsing the Internet on the network, it records it; then will not re-record it for a given period. Thus, software.log maintains a small footprint with useful information. Other logs like the Zeek http.log record every instance.
