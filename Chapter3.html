Which of the following is a key goal of a tactical SIEM?

correct answer: Early detection.

Explanation

Providing early detection is a key goal of a tactical SIEM. The initial compromise is not the end goal of an attacker. Rather, the end goal usually involves something like stealing sensitive data or changing or destroying system integrity, etc. A tactical SIEM will catch early signs of compromise, hopefully enabling defenders to eliminate adversary access early enough that their end goal is never achieved.

Question 3 of 20

Which Windows event log format is stored as a binary XML file?

correct answer: EVTX.

Explanation

EVT and EVTX are the two main log types for Windows systems. Both are binary files, but EVTX adds XML capabilities to events. It allows Windows a granular layout by allowing custom fields, also referred to as properties, in the EventData and UserData sections. This means additional filtering and searching capabilities.

An organization has a business requirement that rules out the use of a third-party agent. A SIEM engineer is tasked with providing application logs to the security analysts for monitoring and analysis. What method can the SIEM engineer use to upload the logs on a Windows file server?

correct answer: Blind drop.

Explanation

The solution for cases that cannot deploy agents is to create a blind drop box; this involves setting up a file server with a share that allows files to be uploaded with write permission only. Once this is in place, a log agent can be installed on the file server to constantly look for and process new files. Then, endpoints can be set to either save these types of logs directly to the file server or have a script that regularly copies existing logs into this location.

Question 5 of 20

Why would a security architect not choose syslog to transport Windows Event Log data?

correct answer: Loss of visibility

Explanation

Syslog is not a smart choice for transporting Windows Event Log data due to visibility loss. Microsoft made a smart decision when it decided to add the XML schema capabilities in EVTX files. It effectively added intelligent, structured logs that contain more data and have more precision and filtering capabilities.

Because EVTX events use XML, an infinite number of fields can be created. This is important to note, as traditional log agents often parse out the most critical fields. This can result in having 50 to 100 Windows fields to search. While this simplifies searching and filtering, it also is blinding, as there are easily over 500 fields that may be necessary for certain techniques to work.

Question 12 of 20

Based on the log output below, what is the severity of the syslog message?

<81>Jun 20 12:25:11 logparse sudo: jdavis : 1 incorrect password attempt ; TTY=pts/1 ; PWD=/var/log ; USER=root ; COMMAND=/bin/su

correct answer: 1.

Explanation

To calculate the severity, you must first determine the facility value. The facility number equals the PRI number divided by eight and rounded down to the nearest whole number.

Facility = PRI / 8 ( ( 81 / 8 = 10.125 )
Facility = 10

The severity number equals the PRI number minus the result of the facility times eight.

Severity = PRI - (Facility * 8) ( ( 81 - (10 * 8) ) = ( 81 - 80 )
Severity = 1

Question 20 of 20

Which of the following Windows logging types is often off by default?

correct answer: Analytic.

Explanation

Under the hood, there are four channel types: admin, operational, analytic, and debug. The purpose of these channels is to describe the type of log they contain. For instance, admin logs are for events that are well documented, and any resolution necessary is described in detail. Analytic and debug are often off by default due to the high volume of events generated when enabled. These channels also frequently require stopping prior to reading the events.
